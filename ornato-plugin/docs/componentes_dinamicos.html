01Filosofia do sistema

Princípio central Uma peça é uma peça porque diz que é, não porque parece ser. Nenhum algoritmo geométrico é a fonte de verdade — o atributo SketchUp é.

O sistema Ornato usa o AttributeDictionary 'Ornato' como fonte de identidade. Cada grupo no modelo declara explicitamente o que é. Isso é equivalente ao wpsisashape = 1 do WPS.

Sem o atributo tipo = "peca", um grupo é invisível para toda a pipeline — não entra na lista de corte, não gera usinagem, não aparece no custo.

Hierarquia de um móvel no SketchUp

Grupo "Balcão 600×850×600mm" [tipo=modulo] ├── Grupo "Lateral Esq" [tipo=peca, role=lateral] ├── Grupo "Lateral Dir" [tipo=peca, role=lateral] ├── Grupo "Base" [tipo=peca, role=base] ├── Grupo "Tampo" [tipo=peca, role=top] ├── Grupo "Traseira" [tipo=peca, role=back_panel] ├── Grupo "Porta" [tipo=peca, role=door] └── Componente "Dobradiça" [tipo=ferragem] ← invisível ao sistema

O grupo-módulo é o container. Ele não entra na lista de corte. Cada filho marcado como peca entra.

03O PieceStamper — identidade de cada peça

O PieceStamper é o sistema de identidade. Equivalente ao wpsisashape = 1 do WPS. Arquivo: library/piece_stamper.rb

Carimbar uma peça

PieceStamper.stamp(group,
  role:      :lateral,
  material: 'MDF18_BrancoTX',
  espessura: 18,
  bordas:   { frente: true, tras: false, topo: true, base: false },
  obs:      'Lateral esquerda'
)

Carimbar um módulo container

PieceStamper.stamp_module(group,
  module_id: 'balcao_simples',
  params:    { 'largura' => 600, 'altura' => 850 },
  nome:      'Balcão Simples'
)

Verificações

PieceStamper.piece?(entity)          # → true/false
PieceStamper.module?(entity)         # → true/false
PieceStamper.tipo(entity)            # → 'peca' | 'modulo' | 'ferragem' | nil
PieceStamper.fully_stamped?(entity)  # → true se todos os campos obrigatórios ok
PieceStamper.read(entity)            # → Hash com todos os atributos
PieceStamper.dimensions(entity)      # → { comprimento:, largura:, espessura: }
PieceStamper.find_pieces(entities)   # → Array de peças

09Ferragens reais — blocos 3D no modelo

Crítico Todo bloco 3D de ferragem DEVE ter set_attribute('Ornato', 'tipo', 'ferragem'). Sem isso, o sistema pode confundi-lo com uma peça de corte e incluí-lo na lista.

Inserindo uma dobradiça 3D

def insert_hinge_3d(parent_group, position_xyz)
  model = Sketchup.active_model

  hinge_path = File.join(Ornato::PLUGIN_DIR,
    'assets', '3d', 'dobradica_blum_clip_top.skp')

  definition = model.definitions.load(hinge_path)

  t = Geom::Transformation.new(Geom::Point3d.new(
    position_xyz[0].mm,
    position_xyz[1].mm,
    position_xyz[2].mm
  ))

  instance = parent_group.entities.add_instance(definition, t)
  instance.name = 'Dobradiça Blum Clip Top'

  # OBRIGATÓRIO: marcar como ferragem
  instance.set_attribute('Ornato', 'tipo',            'ferragem')
  instance.set_attribute('Ornato', 'ferragem_tipo',   'dobradica')
  instance.set_attribute('Ornato', 'ferragem_modelo', 'Blum Clip Top 110°')

  instance
end

13Rebuild dinâmico

O JsonModuleBuilder.rebuild é o equivalente ao "atualizar componente dinâmico" do SketchUp — reconstrói o módulo com novos parâmetros sem precisar recriar do zero.

Fluxo do rebuild

1
Ler module_id do atributo Ornato.module_id do grupo
2
Carregar JSON original da biblioteca
3
Mesclar params antigos com os novos (user tem prioridade)
4
Apagar todos os filhos do grupo (as peças)
5
Reconstruir chamando builder.build(group)
6
Atualizar Ornato.params no grupo

Mudar espessura de 18mm para 25mm

group = Sketchup.active_model.selection.first

if Library::PieceStamper.module?(group)

Fundo encaixado num rasgo nas laterais, base e tampo.

Parafusado

Painel mais grosso fica por trás das laterais.

  • Largura = largura total do módulo
  • Cola + parafusos

23Tutorial: do zero ao módulo legível pelo sistema

O que você vai construir Um Balcão com Porta e Gaveta — modelo completo com dobradiças Blum Clip Top posicionadas corretamente e corrediça oculta Tandem. Ao final o sistema ERP consegue ler todas as peças, gerar lista de corte, plano de furo e custo sem nenhum dado manual.

Este tutorial é linear. Leia do início ao fim na primeira vez. Depois use o índice lateral para consultas rápidas.

Visão geral — os dois fluxos de trabalho

Existem dois caminhos para criar um módulo que o sistema consegue ler. Entenda qual usar em cada situação:

🔨 Fluxo A — SketchUp primeiro (padrão)

  • Você modela as peças manualmente no SketchUp
  • Estampa cada grupo com o PieceStamper via Ruby Console
  • Posiciona dobradiças e corrediças como componentes 3D
  • Salva o projeto como .skp
  • O ERP lê quando solicitado — gera JSON automaticamente

✓ Use para projetos únicos e personalizados

📐 Fluxo B — Template JSON (biblioteca)

  • Você escreve um arquivo .json com fórmulas
  • O JsonModuleBuilder cria o móvel no SketchUp
  • Estampa automático — PieceStamper chamado pelo builder
  • Rebuild dinâmico: mude parâmetros com um comando Ruby
  • Salva o projeto como .skp (igual ao Fluxo A)

✓ Use para módulos repetitivos da biblioteca

Os dois fluxos convergem para o mesmo ponto Em ambos os casos, o resultado final é um .skp com atributos Ornato em cada grupo. O ERP não sabe (nem precisa saber) qual fluxo foi usado — ele lê os atributos, não o histórico. Este tutorial cobre o Fluxo A completo (a mais usada na prática) e depois explica o Fluxo B.

Variações comuns a partir deste módulo

Balcão sem porta

Remova a peça role=door e os parâmetros abertura_*. Não precisa de ferragens_auto.

Balcão 2 portas

Adicione um divisor central (role=divider) e duas peças role=door com "abertura_largura": "{(largura - 2*espessura - espessura) / 2}".

Aéreo (porta passante)

Mude tipo_porta para "passante_desce" e adicione "extensao_passante": 60 nos params. A porta descerá além da base do módulo.

Com gaveta

Adicione peças role=drawer_front, drawer_side, drawer_back, drawer_bottom e ferragem corredica_tandem em ferragens_auto.

Com prateleira

Adicione peça role=shelf com "comprimento": "{largura - 2*espessura - 4}" (folga de 2 mm de cada lado) e "z": "{altura / 2}".

Porta em cores diferentes

Defina "material_frente": "MDF18_Grafite" nos params. A porta e frentes de gaveta puxam material_frente; a caixa usa material_carcaca.

#  HELPER — cole isto uma vez no Ruby Console e use
#  stamp_sel() para estampar a peça selecionada
# ════════════════════════════════════════════════════
def stamp_sel(role, material, bordas = {})
  ent = Sketchup.active_model.selection.first
  unless ent
    puts "⚠ Nenhum grupo selecionado!"
    return
  end
  esp = Ornato::Catalog::MaterialCatalog.instance.thickness(material) rescue 18
  Ornato::Library::PieceStamper.stamp(ent,
    role:      role,
    material:  material,
    espessura: esp,
    bordas:    bordas
  )
  puts "✓ #{ent.name} → role: #{role}  mat: #{material}  esp: #{esp}mm"
end

25. Fórmulas e Expressões Paramétricas

O sistema de expressões permite que as dimensões de cada peça sejam calculadas dinamicamente a partir dos parâmetros do módulo. Em vez de valores fixos, você escreve fórmulas que o plugin avalia no momento de construir ou reconstruir o móvel.

25.1 Sintaxe básica

Qualquer campo largura, altura, profundidade ou posição no JSON de template pode receber uma expressão:

// Valor fixo (número)
"largura": 600

// Expressão — string com {param}
"largura": "{modulo_largura} - 2 * {esp_lat}"

// Expressão com constante mista
"altura": "{modulo_altura} - {esp_topo} - {esp_base} - 4"

Regras de parsing:

RegraExemploResultado
Se o valor for um número puro1818 (float)
Se for string SEM {}"600"600.0
Se for string COM {param}"{mod_larg} - 36"avalia como expressão
Operadores suportados+ - * / ( )precedência normal
Parâmetro desconhecido{xyz}substitui por 0, log de aviso
⚠ Atenção: As chaves {} só funcionam dentro de strings. Se você colocar um número no JSON sem aspas ("largura": 600), ele é tratado como fixo e não é avaliado como expressão.

25.2 Funções disponíveis

Dentro de qualquer expressão você pode chamar as funções abaixo. Elas são avaliadas pelo método eval_dim() do plugin antes de criar a geometria.

FunçãoSintaxeDescriçãoExemplo
max()max(a, b)Retorna o maior entre dois valoresmax({mod_larg} - 36, 200)
min()min(a, b)Retorna o menor entre dois valoresmin({n_prateleiras} * 5, 30)
round()round(a)Arredonda para o inteiro mais próximoround({mod_larg} / 3)
floor()floor(a)Arredonda para baixo (trunca)floor({altura_util} / 32)
ceil()ceil(a)Arredonda para cimaceil({altura_util} / 32)

// prateleira 1: base + espaço

25.3 Expressões condicionais

O parser suporta um operador ternário simplificado usando a notação if(cond, val_true, val_false):

Como o plugin processa as fórmulas em Ruby:

# Em JsonModuleBuilder — método eval_dim(expr, params)
def self.eval_dim(expr, params)
  return expr.to_f if expr.is_a?(Numeric)
  str = expr.to_s

  # Substitui {param} pelos valores numéricos
  str.gsub!(/{(\w+)}/) do |match|
    key = $1
    val = params[key] || params[key.to_sym]
    val ? val.to_f.to_s : '0'
  end

  # Expande funções: max, min, round, floor, ceil
  str.gsub!(/\bmax\(([^,]+),([^)]+)\)/) { [eval($1), eval($2)].max }
  str.gsub!(/\bmin\(([^,]+),([^)]+)\)/) { [eval($1), eval($2)].min }
  str.gsub!(/\bround\(([^)]+)\)/)       { eval($1).round }
  str.gsub!(/\bfloor\(([^)]+)\)/)       { eval($1).to_f.floor }
  str.gsub!(/\bceil\(([^)]+)\)/)        { eval($1).to_f.ceil }

  # Avalia a expressão matemática final
  eval(str).to_f
rescue => e
  ORNATO_LOG.warn "eval_dim falhou para '#{expr}': #{e.message}"
  0.0
end

27. Fitas de Borda

Fita de borda (edge banding) é a tira de material — ABS, PVC ou madeira — colada nas arestas expostas das peças de MDF/MDP. Ela protege o miolo, dá acabamento e afeta diretamente as dimensões de corte. O ERP desconta automaticamente a espessura da fita no plano de corte — você não precisa compensar manualmente.

⚠ Impacto nas dimensões: Se uma lateral recebe fita de 1mm na aresta do topo, a peça é cortada com altura − 1mm. O atributo bordas diz ao ERP quais arestas têm fita e de qual espessura, e ele faz o desconto sozinho na lista de corte.

Há duas formas de declarar bordas em uma peça do template JSON:

Forma 1 — Booleano simples (usa a fita padrão do módulo definida em fita_padrao):

{
  "params": {
    "fita_padrao":   "BOR_04x22_Branco",
    "fita_frentes":  "BOR_10x22_Branco"
  },
  "pieces": [
    {
      "id":      "lateral_esq",

      "bordas":  { "frente": true, "topo": true, "base": false, "tras": false }
    },
    {
      "id":      "porta",

      "bordas":  { "frente": true, "topo": true, "base": true, "tras": true },
      "fita_override": "{fita_frentes}"
    }
  ]
}

Forma 2 — Código explícito por aresta (controle total por peça):

{
  "pieces": [
    {
      "id":   "lateral_esq",
      "role": "lateral",
      "bordas": {
        "frente": "BOR_10x22_Branco",
        "topo":   "BOR_04x22_Branco",
        "base":   false,
        "tras":   false
      }
    }
  ]
}

Como o ERP usa esse atributo para gerar o plano de corte:

Peça: Lateral Esquerda  |  Nominal: 700mm × 560mm × 18mm
bordas: { frente: "BOR_10x22", topo: "BOR_04x22" }

Cálculo automático ERP:
  Altura de corte = 700mm − 1.0mm (fita topo) = 699mm
  Largura de corte = 560mm (sem fita nas bordas de profundidade)

Plano de corte gerado:
  → CORTAR: 699 × 560 × 18mm
  → COLAR FITA: frente (1mm×22mm) + topo (0.4mm×22mm)
  → Rebarbar e lixar

27.7 Trocar tipo de fita sem reconstruir

Como fita de borda é um atributo (não geometria), você pode trocá-la em todo o módulo com repaint() — sem rebuild, sem perder posição das peças:

modulo = Sketchup.active_model.selection.first

JsonModuleBuilder.repaint(modulo, {
  'fita_padrao'  => 'BOR_10x22_Branco',
  'fita_frentes' => 'BOR_20x22_Branco'  # 2mm nas portas e frentes
})

# Depois disso:
# - As peças com role lateral/base/topo → fita 1mm
# - As peças com role door/drawer_front → fita 2mm
# - O ERP vai recalcular o plano de corte com os novos descontos
⚠ Atenção no desconto duplo: Se uma peça tem fita em ambos os lados opostos (ex: fita no topo e na base de uma divisória), o ERP desconta os dois. Para divisória com 700mm de altura e fita 1mm em topo e base: corte = 700 − 1 − 1 = 698mm.
